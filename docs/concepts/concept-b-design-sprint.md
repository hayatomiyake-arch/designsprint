# Concept B: Design Sprint（デザインスプリント）— 実践ワークショップ型

## コアコンセプト
> 「考えて、作って、壊して、また作る。最速で設計力を鍛える」

実務のデザインプロセスを凝縮。自分で指示を出し、結果を見て、レビューを受け、
改善する——このサイクルを高速で回すことで「設計の筋力」をつける。

## 学習哲学
- 手を動かす（指示を出す）→ 結果を見る → フィードバックを受ける → 改善する
- 最初から「作る側」として参加する
- 失敗からの学びを重視（最大3ラウンドの改善機会）

## 体験フロー（1セッション = 約15〜20分）

### Step 1: ブリーフを受ける
ユーザーの状況とゴール + 制約条件がカード形式で提示される。
例）「引っ越したばかりの30代会社員。近所の美味しいランチを探したい」
制約：モバイルファースト、メイン操作は3タップ以内

### Step 2: 設計指示を書く
受講者が「どんな画面を作りたいか」をテキストで指示。
- 何を画面に配置するか
- どんな優先順位で情報を見せるか
- ユーザーの行動をどう導くか

### Step 3: AIが画面を生成
指示に基づき、AIが1パターンの画面を生成。

### Step 4: 自動レビュー（Round 1）
UIデザイナー + UXデザイナーの2視点でレビューが表示される。
Good Point / Development Point + 具体的な改善ヒント。

### Step 5: 改善ラウンド（最大3回）
- レビューを読む → 改善方針をテキスト入力 → AIが再生成 → 再レビュー
- 各ラウンドで前回との差分がハイライトされる
- スコアの推移が可視化される

### Step 6: 最終スコア
3ラウンドの成長曲線 + 最終評価 + 学びのまとめ

## UI方向性
- IDE/ワークスペース風の3カラムレイアウト
  - 左: お題ブリーフ + 指示入力
  - 中央: 生成されたプレビュー
  - 右: レビュー + スコア
- 明るくエネルギッシュな配色
- プログレスバーでラウンド進行を可視化

## このコンセプトの強み
- 実務に直結する「指示→生成→改善」のサイクル
- 自分で考えて作るので能動的
- 改善の過程が可視化されるのでモチベーション維持しやすい

## このコンセプトの懸念
- 初心者は「何を指示すればいいか分からない」状態になりやすい
- 1セッションの時間がやや長い
- テキスト入力のハードルがある

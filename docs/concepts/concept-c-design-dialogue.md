# Concept C: Design Dialogue（デザイン対話）— ソクラテス式対話型

## コアコンセプト
> 「答えを教わるのではなく、問いを通じて自分で気づく」

ソクラテス式問答法をデザイン学習に応用。AIが「問い」を投げかけ、
受講者が考え、答えることで「デザイン思考」そのものを鍛える。
画面を作る前の「思考のプロセス」を重視。

## 学習哲学
- 問われる → 考える → 言語化する → 形にする → 振り返る
- 「何を作るか」の前に「なぜそう作るか」を徹底的に考える
- 対話を通じて、自分の中にあるデザイン直感を引き出す

## 体験フロー（1セッション = 約15〜20分）

### Step 1: シナリオが提示される
ユーザーの状況とゴールがストーリー形式で提示。
例）「田中さん（32歳）は3日前に引っ越してきたばかり。今日のランチ、
何を食べようか迷っている。周りに知り合いもいないし、
土地勘もない。せめて美味しいものを食べて気分を上げたい——」

### Step 2: AIからの問い（3〜5問）
AIが段階的に思考を深める質問を出す：
1. 「田中さんが最初に知りたい情報は何だと思いますか？」
2. 「その情報を、どんな形で見せたら最もわかりやすいですか？」
3. 「田中さんは"決断"をどう後押しされたいと思いますか？」
4. 「スマホの小さな画面で、最も目立たせるべき要素は？」

各問いに受講者がテキストで回答 → AIが回答を踏まえて次の問いを生成。

### Step 3: 思考の整理
対話の内容がデザイン要件として自動整理される：
- ユーザーの本質的ニーズ
- 情報の優先順位
- UI上の重要ポイント

### Step 4: 画面の自動生成
対話から導かれた要件に基づき、AIが画面を生成。
「あなたの思考がこの画面になりました」という体験。

### Step 5: 振り返りレビュー
- 自分の回答と生成された画面の対応関係を確認
- UIデザイナー / UXデザイナー視点のフィードバック
- 「もし別の回答をしていたら…」のif分岐も見せる

### Step 6: 改善 + 最終評価
対話の気づきを踏まえて改善指示 → 最終レビュー

## UI方向性
- チャット/メッセンジャー風のUI（LINEやiMessage的）
- 左側にストーリー、右側が対話エリア
- 対話の流れが自然に画面生成につながるトランジション
- 温かみのある配色、1対1の対話感
- 思考のマップが視覚化される要素あり

## このコンセプトの強み
- 「デザイン思考」そのものが身につく（表面的なスキルでなく本質）
- 対話形式なので取り組みやすい（テキスト入力のハードルが低い）
- 「なぜ」を常に考えるので、応用が効く力がつく
- 生成AI時代の「AIへの指示力」も同時に鍛えられる

## このコンセプトの懸念
- 思考が深まるまでに時間がかかる
- 「早く作りたい」タイプの人には回りくどく感じるかも
- 対話の質がAIの質問力に大きく依存する

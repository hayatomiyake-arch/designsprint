**フォーカス原則:** {{PRINCIPLE}}

## 提出されたデザイン（HTML）

```html
{{SUBMITTED_HTML}}
```

## 指示

同じお題・同じ制約に対して、**まったく異なるレイアウト戦略** で2つの別解を生成してください。

### 別解 A: 別のレイアウトアプローチ
- 提出デザインとは異なるレイアウト構造（例: カード型 → リスト型、縦スクロール → タブ切替 など）
- 同じ情報を異なる視覚的優先度で表現

### 別解 B: 別のUXアプローチ
- ユーザー体験の組み立て方が異なる（例: 検索優先 → レコメンド優先、一覧表示 → ステップ形式 など）
- 同じゴールを別の情報設計で達成

## 出力形式

**必ず以下のJSON形式のみを出力してください。**

```json
{
  "altA": {
    "label": "別解Aの短いラベル（例: リスト型レイアウト）",
    "rationale": "この設計アプローチの意図（2-3文）",
    "strengths": ["強み1", "強み2"],
    "html": "<!DOCTYPE html>...完全なHTML..."
  },
  "altB": {
    "label": "別解Bの短いラベル（例: タブ切替型）",
    "rationale": "この設計アプローチの意図（2-3文）",
    "strengths": ["強み1", "強み2"],
    "html": "<!DOCTYPE html>...完全なHTML..."
  },
  "comparison": [
    {
      "aspect": "比較観点1（例: レイアウト）",
      "mine": "提出デザインの特徴",
      "altA": "別解Aの特徴",
      "altB": "別解Bの特徴"
    },
    {
      "aspect": "比較観点2（例: 情報優先度）",
      "mine": "提出デザインの特徴",
      "altA": "別解Aの特徴",
      "altB": "別解Bの特徴"
    },
    {
      "aspect": "比較観点3（例: 強み）",
      "mine": "提出デザインの強み",
      "altA": "別解Aの強み",
      "altB": "別解Bの強み"
    }
  ]
}
```

## ルール

1. 別解のHTMLは生成プロンプトと同じ仕様（340px幅、日本語、外部リソース不使用、完全HTML）
2. 別解は提出デザインより「良い」必要はない。**異なるアプローチ** であることが重要
3. 各別解の rationale は「なぜこの設計にしたか」を説明。学びのポイントになる
4. comparison は3-4行で、デザイン判断の違いを浮き彫りにする
5. JSONのみ出力。バッククォートやマークダウン記法で囲まない

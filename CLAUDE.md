# Proto Review

## プロジェクト概要
Claude Code（Cowork）で開発したプロトタイプを一覧プレビューし、UI/UX評価で品質を磨けるサービス。
前プロジェクト（DesignSprint: UI/UXトレーニングサービス）のエッセンスを引き継ぎ、新サービスとして再構築。

## 開発フェーズ
- **Phase 1（現在）**: コンセプト検証 — 3パターンのストーリーボード → コア体験プロトタイプ
- **Phase 2**: MVP — 最小機能セットで検証
- **Phase 3**: 本開発 — 要求が固まってから着手

## 技術スタック
- Phase 1: HTML / CSS / Vanilla JS（ビルドツール不要）
- Phase 2以降: Cloudflare Pages + Workers / 必要に応じてフレームワーク導入

## プロジェクト構造
```
proto-review/             # 新サービス Proto Review
  docs/                   # 設計ドキュメント
    concept.md            # サービスコンセプト
    prototype-plan.md     # プロトタイプ計画（3パターン）
  prototypes/             # UIプロトタイプ（HTML/CSS/JS）
    pattern-a/            # パターンA: ギャラリー型
    pattern-b/            # パターンB: ワークスペース型
    pattern-c/            # パターンC: ステップ型
  learnings-from-designsprint.md  # 前プロジェクトからの学び
```

## コマンド
- Phase 1: 該当なし（静的HTML、ブラウザで直接開く）
- Phase 2以降で追加予定

## コーディング規約
- 変数名・関数名・コメント・ドキュメント: 日本語OK
- UI上のラベル・テキスト: 日本語
- コード構造: シンプル第一。過剰な抽象化をしない
- コミットメッセージ: 日本語

## 設計方針
- アジャイル: コンセプト → 要求 → 要件 → プロトタイプ → 磨き込み → 本開発
- 3パターンのコンセプトプロトタイプを作り、A/Bテスト的に方針を固める
- コア体験を段階的にブラッシュアップ
- AI連携はコスト最適化を重視（無料枠 → 段階的アップグレード）

## レビュー基準
- UIデザイナー視点: 視覚的階層、一貫性、アクセシビリティ
- UXデザイナー視点: ユーザーフロー、情報設計、認知負荷
- 評価: Good point(最大3つ) / Development point(0〜3つ、無理に出さない)
